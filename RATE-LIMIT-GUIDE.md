# 智谱 AI 速率限制问题解决指南

## 问题说明

当你看到错误消息：`429 您的账户已达到速率限制，请您控制请求频率`

这表示你的 API 请求过于频繁，或者账户达到了使用限制。

---

## 🔍 常见原因

### 1. 请求过于频繁
- 短时间内发送了太多请求
- 智谱 AI 免费账户有 QPM（每分钟请求数）限制
- 通常限制为：2-5 次/分钟

### 2. 免费额度用完
- 新用户免费额度：1000 万 tokens
- 可在控制台查看剩余额度

### 3. 并发请求超限
- 同时发起多个请求
- 超过账户的并发限制

---

## ✅ 解决方案

### 方案 1：等待重试（最简单）

**操作步骤**：
1. 等待 **1-2 分钟**
2. 刷新页面
3. 重新选择文件生成摘要

### 方案 2：检查账户额度

**操作步骤**：
1. 访问：https://open.bigmodel.cn/usercenter/proj-mgmt/apikeys
2. 登录你的账户
3. 查看 **资源包** 或 **余额**
4. 如果用完，需要充值或等待额度重置

### 方案 3：升级账户

**操作步骤**：
1. 访问智谱 AI 控制台
2. 选择付费套餐
3. 获得更高的速率限制：
   - 更高的 QPM（每分钟请求数）
   - 更多的 token 额度
   - 更快的响应速度

### 方案 4：优化使用方式

**建议**：
- ✅ 避免频繁点击"生成摘要"按钮
- ✅ 间隔至少 **15-30 秒**再次请求
- ✅ 不要同时打开多个标签页测试
- ✅ 选择较短的文档进行测试

---

## 🎯 智谱 AI 速率限制详情

### 免费账户限制（参考值）
- **QPM (每分钟请求)**：2-5 次
- **RPD (每天请求)**：100-1000 次
- **Token 限制**：1000 万（一次性）
- **并发数**：1-2

### 付费账户优势
- **QPM**：10-100+ 次
- **RPD**：无限制或很高
- **Token**：按需充值
- **并发数**：5-10+

## 🔧 技术优化（仅供参考）

如果你是开发者，可以在代码中添加以下优化：

### 1. 添加请求节流
```typescript
// 限制请求频率
let lastRequestTime = 0;
const MIN_INTERVAL = 15000; // 15 秒

async function generateSummary() {
  const now = Date.now();
  if (now - lastRequestTime < MIN_INTERVAL) {
    alert(`请等待 ${Math.ceil((MIN_INTERVAL - (now - lastRequestTime)) / 1000)} 秒后再试`);
    return;
  }
  lastRequestTime = now;
  // 继续执行...
}
```

### 2. 添加重试机制
```typescript
async function retryRequest(fn: Function, maxRetries = 3, delay = 2000) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error: any) {
      if (error.status === 429 && i < maxRetries - 1) {
        await new Promise(resolve => setTimeout(resolve, delay * (i + 1)));
        continue;
      }
      throw error;
    }
  }
}
```

### 3. 添加加载提示
在按钮上显示冷却时间，防止用户频繁点击。

## 🆘 常见问题

### Q1: 为什么显示已达到速率限制，但我才第一次使用？
**A**: 可能是：
- 之前测试时已经用过几次
- 账户在其他地方被使用
- 系统检测到异常请求

### Q2: 充值后还是有限制吗？
**A**: 付费账户仍有速率限制，但限制会大大提高（通常 10-100 倍）。

### Q3: 可以换其他 AI 服务吗？
**A**: 可以，参考 [AI-API-SETUP.md](AI-API-SETUP.md) 了解其他选项：
- 阿里云通义千问
- 百度文心一言
- DeepSeek（价格更便宜）

### Q4: 测试项目怎么办？
**A**: 建议：
- 使用较短的测试文档
- 间隔 1-2 分钟测试一次
- 或者申请多个测试账号
- 考虑使用付费账户（学生可能有优惠）
